# AI 時事快報 — 架構構思提案 v0.1

> 目標：讓 geek 每天 3–5 分鐘掌握 AI / LLM / 前沿科技重點。不洋洋灑灑，只給重點，展開才有細節。

---

## 一、整體架構（建議）

```
┌─────────────┐   ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│  來源層      │ → │  生成層       │ → │  儲存層       │ → │  呈現層       │
│  RSS/API    │   │  LLM 摘要策展 │   │  Git 即資料庫 │   │  GitHub Pages│
└─────────────┘   └──────────────┘   └──────────────┘   └──────────────┘
                         ↑
                  每日排程自動觸發（GitHub Actions cron 或 Cowork 排程）
```

核心設計原則：**不需要資料庫、不需要伺服器、不需要重爬蟲。**

- **爬蟲 → 用 RSS/API 取代**：主要來源幾乎都有 RSS 或公開 API，比爬 HTML 穩定十倍，也不會被擋。
- **資料庫 → 用 Git 取代**：每日報就是一個 JSON/Markdown 檔，commit 進 repo。Git history 天然就是封存與版本紀錄，免費、可靠、可回溯。
- **伺服器 → 用靜態網站取代**：GitHub Pages 免費託管，純前端 JS 做語言切換、深淺色、收合。

## 二、來源層

第一批建議來源（皆有 RSS/API，之後可增減）：

| 類別 | 來源 |
|---|---|
| 官方公告 | Anthropic / OpenAI / Google DeepMind / Meta AI blog |
| 研究論文 | arXiv cs.CL / cs.AI / cs.LG（每日新上架） |
| 開源社群 | Hugging Face blog、GitHub Trending |
| 社群訊號 | Hacker News（AI 相關高分文）、Reddit r/LocalLLaMA、r/MachineLearning |
| 策展電子報 | Import AI、The Batch、Simon Willison blog |
| 產業新聞 | The Verge AI、TechCrunch AI、Ars Technica |

社群訊號（HN 分數、Reddit upvotes）可當作「重要性排序」的依據——等於借用社群的集體判斷來過濾雜訊。

## 三、生成層（每日流程）

每天固定時間（建議台北 07:00）自動執行：

1. 抓取過去 24 小時所有來源新項目
2. 去重、依社群熱度與來源權重排序
3. LLM 一次生成雙語（正體中文 + English）結構化日報 JSON
4. Commit 到 repo → GitHub Pages 自動更新

**週一特別版**：多跑一步「回顧本週 7 天日報」，先產出一週綜觀（趨勢、本週三大事），再接當日重點。

### 兩種自動化模式（關鍵決策點）

| | A. GitHub Actions + Claude API | B. Cowork 排程任務 |
|---|---|---|
| 運作方式 | 雲端 cron，全自動，與你的電腦無關 | Claude 每日在你電腦上執行、research、寫檔、push |
| 成本 | API 費用約 US$3–10/月 | 含在訂閱內，無額外費用 |
| 可靠度 | 高（雲端獨立運行） | 需要電腦開機、app 運行 |
| 品質 | 依 pipeline 設計 | 可用 web search 做更深入的判斷與查證 |

建議：**先用 B 快速起步驗證格式與品味，穩定後再遷移到 A 全自動化。**

## 四、儲存層（Git 即資料庫）

```
ai-daily/
├── reports/
│   └── 2026/07/
│       ├── 2026-07-06.json      ← 每日一檔，含雙語內容
│       └── 2026-07-07.json      ← 週一檔多一個 weekly 區塊
├── index.html                    ← 單頁應用（讀取 JSON 渲染）
├── archive.json                  ← 日報索引（自動維護）
└── .github/workflows/daily.yml   ← 排程（模式 A 時）
```

日報 JSON 結構（雙語同檔，前端切換）：

```json
{
  "date": "2026-07-07",
  "type": "daily",
  "sections": [{
    "id": "headlines",
    "title": { "zh": "今日頭條", "en": "Top Stories" },
    "items": [{
      "headline": { "zh": "…一句話標題", "en": "…" },
      "detail":   { "zh": "…2-3 句展開", "en": "…" },
      "source": "https://…",
      "tags": ["llm", "anthropic"]
    }]
  }]
}
```

## 五、呈現層（UI 設計）

- **收合優先**：預設只顯示一句話標題列表，點擊展開 2–3 句細節 + 原文連結。整份日報一眼掃完 < 1 分鐘。
- **固定欄目**（收合區塊，可全展/全收）：
  1. 今日頭條（3 則以內）
  2. 模型與研究
  3. 開源與工具
  4. 產品與應用
  5. 產業與政策
  6. 一句話快訊（不展開，純掃描用）
- 正體中文 ⇄ English 一鍵切換（同一份 JSON，零延遲）
- 深色 ⇄ 淺色模式（跟隨系統 + 手動覆蓋，記憶偏好）
- 封存頁：依月份列出歷史日報，點擊即讀；週報有特別標記
- 行動裝置友善（早上通勤手機看）

## 六、分階段實作

| 階段 | 內容 | 產出 |
|---|---|---|
| P1 骨架 | repo + 靜態網站 + 手動產第一份日報驗證格式 | 可看的網站 |
| P2 自動化 | 接上排程（模式 A 或 B）+ 週報邏輯 + 封存索引 | 每日自動更新 |
| P3 進化 | 標籤篩選、全文搜尋、RSS 輸出、來源權重調校 | 依使用體驗迭代 |

## 七、已定決策（2026-07-07）

1. 自動化模式：**全上雲，$0/月**。cron-job.org（免費）準點觸發 GitHub Actions `workflow_dispatch`（repo 內 `schedule` 留作備援，因 GitHub cron 常延遲/跳過）→ 拉 RSS/免費 API → **Gemini 3 Flash 免費層**（1,500 req/天，我們只用 1–3 次）生成雙語 JSON → commit → Pages。生成腳本 model-agnostic，可隨時換回 Claude API。注意：Gemini 專案不可啟用 billing，否則失去免費層。
2. GitHub repo：**公開**。
3. 內容口味：**偏應用與工具**（開源工具、agent 應用、實作導向為主），保留產業與研究動態追蹤。
   → 欄目權重調整：開源與工具、產品與應用排前；模型與研究精選；產業/政策多入快訊。
